# Rap-LI：風險感知的 LLM 推論隱私保護框架

---

## 📌 專案簡介

大型語言模型（LLM）如 ChatGPT 已廣泛應用於各種任務，但使用者提示（prompt）常包含高度敏感的個人資訊（PII，例如姓名、年齡、病史）或上下文隱私資訊。若未妥善處理，這些資料在送往雲端 LLM 服務時極易被收集、用於訓練，或遭攻擊者推斷，導致嚴重隱私外洩。

本專案基於 IEEE Transactions on Information Forensics and Security 論文 **《Risk-Aware Privacy Preservation for LLM Inference》**，提出 **Rap-LI**（Risk-Aware Privacy Preservation for LLM Inference）框架，實現**風險感知**的本地差分隱私保護，在提供嚴格形式化隱私保證的同時，大幅提升隱私-效用平衡。

---

## ⚠️ 核心問題與動機

現有本地差分隱私（Local Differential Privacy, LDP）文字消毒機制雖具輕量、plug-and-play 優勢，但普遍採用**統一隱私預算（uniform ε）**，忽略文字中不同部分的敏感度差異（例如「飲食」 vs. 「疾病」）。

論文實證顯示，即使在嚴格隱私預算 **ε=0.1** 下，敏感資訊洩漏率仍高達 **71.74%**。統一消毒也常造成語意嚴重失真，影響下游 LLM 任務效用，形成隱私-效用權衡困境。其他方法（如簡單遮罩或 LLM 重寫）也存在保護不足或資源消耗過高的問題。

**研究動機**：提出**全球首個風險感知（risk-aware）的 LDP 框架**，實現上下文感知、個人化保護，同時兼顧嚴格隱私保證與實用效用，填補 LDP 在 LLM 推論場景的關鍵空白。

---

## ✨ 主要成果與創新

Rap-LI 框架主要創新包含：

1. **風險識別與個人化標記**  
   自動偵測提示中的隱私風險，並根據敏感度進行個人化標記，映射到**異質隱私預算（heterogeneous ε）**，突破傳統統一預算限制。

2. **風險感知的 LDP 機制**  
   - 提出新型 **token-level ε-LDP** 與 **sentence-level LDP** 形式化定義  
   - 建構基於語意相似性的有界候選 token 空間，使用指數機制進行風險感知抽樣  
   - 提供嚴格的 token 與句子層級隱私保證

3. **提示工程緩解效用損失**  
   整合任務感知提示工程，提升消毒後提示對下游 LLM 的穩健性

### 實驗成果（多下游任務、資料集與攻擊情境驗證）

- **隱私保護大幅提升**：相較於效用相當的基準方法，敏感資訊洩漏平均降低 **51.68%**
- **效用保留優秀**：在語意相關性、任務準確率等方面維持競爭力，甚至優於部分強基準
- **高度實用性**：plug-and-play、無需訓練、支援互動式風險調整

作者提供完整**互動 Demo**（類 ChatGPT 的隱私保護聊天系統），包含側邊比較、實體還原等功能。框架已在多種黑盒 LLM 上測試，證明廣泛適用性。

---

## 🔍 分析與洞見

### 優勢與創新點
- **風險感知**為核心突破，實現「**按需保護**」：高敏感 token 給予更強保護，低敏感部分保留更多語意，有效解決「一刀切」問題。
- 形式化保證延伸至句子層級，強化對上下文推斷攻擊的防禦。
- 實務導向設計 + 提示工程，兼顧理論嚴謹性與工程可部署性。
- 詳細討論邊緣案例（如高敏感資料在低 ε 下的保護、不同任務權衡、具背景知識的攻擊者等），展現穩健性。

### 潛在限制與權衡
- 風險識別階段引入少量計算開銷（仍屬輕量級）
- 過於保守的個人化標記可能影響高度依賴上下文的任務
- 極端低 ε 下效用仍受影響，需依應用場景調整（醫療、金融等高隱私領域特別適合）
- 未來可擴展至多語言、多模態或動態風險評估

### 更廣泛意義
此工作凸顯 LLM 時代隱私保護需從「靜態消毒」轉向「動態、上下文感知」模式，為後續個人化差分隱私研究奠定基礎，也為產業提供降低對雲端信任依賴的用戶端解決方案，具有重要政策與部署意義。

---

## 📝 結論

Rap-LI 成功彌補現有 LDP 機制在 LLM 推論中的關鍵漏洞，實現更精準、高效的隱私保護，並大幅提升隱私-效用平衡。作為首個風險感知 LDP 框架，它為上下文感知、個人化隱私保護技術開創新方向。

在 LLM 快速普及的背景下，本論文強調**「隱私優先、風險自適應」**的設計哲學，對開發隱私增強技術與建構可信 AI 系統具有重要參考價值。
